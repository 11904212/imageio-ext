The Image I/O-Ext project extends the Java Sun's ImageI/O framework capabilities by providing access to new raster formats directly or through the integration of other frameworks and libraries, like GDAL, JMAgick (experimental) and Kakadu (experimental).

Main Features

The core module of the project is the gdalframework module which provide a bridge between the Java imaeio framework and the Geospatial Data Abstraction Library (GDAL) via SWIG's generated Java bindings to provide support for a reach set of data formats. Here below a brief list of the current supported formats si provided. Note that for each of these formats an imageio-ext plugin is provided which integrates into the standard SUN imageio SPI mechanism.

Utility modules

The Image I/O-Ext project provides some additional utility modules as desribe here below.

  • imagereadmt provides a specific ImageRead operation implementation with support for multithreading.

  • customstreams contains a wide set of extended input/output streams

  • jhdfaccess revisited version of the ncsa JHDF libraries which provide access to HDF format. It currently provides onlt read access to the HDF4 format but write access capabilities are under development.

Feedback and Discussion

Five main Image I/O-Ext mailing lists are available in this project: dev 'at' imageio-ext.dev.java.net, announce 'at' imageio-ext.dev.java.net, issues 'at' imageio-ext.dev.java.net, commits 'at' imageio-ext.dev.java.net, and users 'at' imageio-ext.dev.java.net (replace 'at' with @). Click here to subscribe to any of these lists.

The dev list is a list for developers to discuss project development issues/artifacts and requirements.
The announce list is a low-volume moderated mailing list for announcing Image I/O-Ext news and information.
The issues list is for automated issues tracking and the commits list is for automated subversion commits messages.
Lastly, the users is a general discussion list for the projects end users.
